# ระบบกู้ยืมเงินรายย่อย (Micro Loan Management System)

ระบบจัดการสัญญากู้ยืมเงินรายย่อย ฝั่ง Backend เป็น Express + MySQL, ฝั่ง Frontend เป็น React (Vite) SPA

## โครงสร้างโปรเจกต์

```
.
├── server.js              # Express entry point
├── db.js                  # MySQL connection pool (mysql2)
├── routes/                # API routes (/api/*)
│   ├── auth.js             # admin login/register/setup, admins CRUD, profile, bank-info
│   ├── customer.js         # customer login/register, loans, request-loan, pay
│   ├── customers.js        # admin: customer CRUD
│   ├── loans.js            # admin: loan list/detail/create/status, contract print
│   ├── payments.js         # payment submit/approve
│   └── _shared/schedule.js # shared calcPMT/generateSchedule (amortization)
├── uploads/                # uploaded payment slip images (multer)
├── public/                 # served statically by Express — built React app output
│   ├── index.html           # SPA shell (generated by `vite build`)
│   └── assets/               # built JS/CSS bundles (generated)
├── client/                 # React frontend SOURCE — edit here, not public/
│   ├── src/
│   │   ├── pages/             # route-level page components (admin/, customer/)
│   │   ├── layouts/            # AdminLayout (sidebar shell)
│   │   ├── components/          # Modal, StatusBadge, ProgressBar, etc.
│   │   ├── context/              # AdminAuthContext, CustomerAuthContext, ToastContext
│   │   ├── routes/                # RequireAdmin / RequireSuperAdmin / RequireCustomer guards
│   │   ├── lib/                    # api.js (fetch wrapper), format.js, loan.js (calcPMT)
│   │   └── styles/style.css         # all CSS (admin + customer themes)
│   └── vite.config.js       # dev proxy (/api, /uploads -> :3000) + build outDir ../public
├── borrow_money.sql        # DB schema + seed data dump
└── .env                    # local config (not committed)
```

**สำคัญ:** ห้ามแก้ไฟล์ใน `public/` ตรงๆ — มันถูก generate ทับทุกครั้งที่รัน `npm run build`. แก้ที่ `client/src/` แทน

## ติดตั้งครั้งแรก

### 1. ติดตั้ง dependency ทั้งสองส่วน

```bash
npm install              # root (Express)
npm install --prefix client   # หรือ cd client && npm install
```

### 2. ตั้งค่าฐานข้อมูล MySQL

ต้องมี MySQL/MariaDB รันอยู่ที่เครื่อง (ค่า default ใช้ `localhost:3306`, user `root`, ไม่มีรหัสผ่าน)

สร้างฐานข้อมูลและ import schema + seed data:

```bash
mysql -u root -e "CREATE DATABASE IF NOT EXISTS borrow_money"
mysql -u root borrow_money < borrow_money.sql
```

(หรือใช้ phpMyAdmin/Workbench import ไฟล์ `borrow_money.sql` ก็ได้)

### 3. สร้างไฟล์ `.env` ที่ root โปรเจกต์

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=
DB_NAME=borrow_money
PORT=3000
SESSION_SECRET=เปลี่ยนเป็นค่าสุ่มของจริง
```

## รันตอนพัฒนา (Dev)

```bash
npm run dev
```

คำสั่งนี้รัน **Express + Vite พร้อมกัน** (ผ่าน `concurrently`):

| Server | Path | หน้าที่ |
|---|---|---|
| Express | `http://localhost:3000` | API (`/api/*`) + serve `uploads/` |
| Vite dev server | `http://localhost:5173` (หรือพอร์ตถัดไปถ้าชนกัน เช่น 5174) | React app + hot reload |

**เปิดเบราว์เซอร์ที่พอร์ตของ Vite** (5173/5174) ไม่ใช่ 3000 — Vite จะ proxy request `/api/*` และ `/uploads/*` ไปที่ Express ให้อัตโนมัติ (ดูได้ใน `client/vite.config.js`)

รันแยกส่วนได้ด้วย (เผื่อ debug):

```bash
npm run dev:server    # Express อย่างเดียว (:3000)
npm run dev:client    # Vite อย่างเดียว (:5173)
```

### ตั้งค่าแอดมินคนแรก

ถ้ายังไม่มีแอดมินในระบบ ให้เปิด `http://localhost:5173/setup` กรอกฟอร์มสร้างบัญชีแอดมิน (ทำได้แค่ครั้งเดียวตอนยังไม่มีแอดมินเลย)

ถ้า import `borrow_money.sql` มาแล้วจะมีแอดมิน seed ไว้ให้แล้ว (`username: admin`, `password: admin123`)

## Build สำหรับ Production

```bash
npm run build    # vite build -> เขียนผลลัพธ์ทับ public/ ทั้งโฟลเดอร์
npm start        # node server.js (serve build จาก public/, ไม่มี Vite/proxy แล้ว)
```

เปิด `http://localhost:3000` ได้ตรงๆ (ไม่ต้องผ่านพอร์ต Vite อีกต่อไป)

⚠️ `npm run build` ใช้ `emptyOutDir: true` — มันจะ**ลบไฟล์ทุกไฟล์ใน `public/`** ก่อนเขียนของใหม่ทับ ห้ามวางไฟล์อื่นที่อยากเก็บไว้ใน `public/` ตรงๆ

## URL หลักของแอป

| Path | คำอธิบาย |
|---|---|
| `/login` | เข้าสู่ระบบ/สมัครสมาชิก (รวม admin + ลูกหนี้ในหน้าเดียว, มีปุ่มสลับโหมด) |
| `/customer/login` | ทางลัดไปหน้า login โหมดลูกหนี้ (component เดียวกับ `/login`) |
| `/setup` | ตั้งค่าแอดมินคนแรก (ใช้ได้แค่ตอนยังไม่มีแอดมินในระบบ) |
| `/dashboard` | แดชบอร์ดแอดมิน |
| `/customers`, `/loans`, `/payments` | จัดการลูกหนี้ / สัญญา / การชำระเงิน (แอดมิน) |
| `/loans/:id` | รายละเอียดสัญญา (แอดมิน) |
| `/profile`, `/admins` | เฉพาะ Super Admin (id=1) |
| `/customer` | พอร์ทัลลูกหนี้ — ดูสัญญาของตัวเอง |
| `/customer/loans/:id` | รายละเอียดสัญญา + ชำระเงิน (ลูกหนี้) |

URL เก่าแบบ `*.html?id=` (เช่น `/loan-detail.html?id=5`) ยังใช้งานได้ — Express จะ redirect ไป path ใหม่ให้อัตโนมัติ (ดูใน `server.js`)

## คำสั่งที่ใช้บ่อย

| คำสั่ง | ใช้ทำอะไร |
|---|---|
| `npm run dev` | dev mode เต็มรูปแบบ (Express + Vite, hot reload) |
| `npm run build` | build React ทับ `public/` — **ทำตอนจะ deploy เท่านั้น** |
| `npm start` | รัน production server (ต้อง build ก่อน) |
| `npm install --prefix client <pkg>` | เพิ่ม dependency ฝั่ง frontend |

## หมายเหตุ

- Session ของแอดมินกับลูกหนี้เป็นคนละชุดกัน (`req.session.admin` / `req.session.customer`) — เบราว์เซอร์เดียวสามารถ login เป็นทั้งแอดมินและลูกหนี้พร้อมกันได้ (คนละแท็บ)
- ไฟล์สลิปการโอนเงินถูกเก็บใน `uploads/slips/` (ไม่ได้อยู่ใน `public/` จะไม่ถูกลบตอน build)
- วงเงินกู้สูงสุด 50,000 บาท, ดอกเบี้ยสูงสุด 15%/ปี (ตาม ป.พ.พ. มาตรา 654) — เป็น validation ที่ฝังอยู่ใน `routes/loans.js` และ `routes/customer.js`
